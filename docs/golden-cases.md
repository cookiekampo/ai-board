# Golden Cases

このファイルは、AI会議室の重要な回帰テスト用ケースを記録する開発メモです。
現時点ではUIには表示しません。

## GC-DRP-KAMPO-FIBROMYALGIA-001

### 対象モード

Deep Researchプロンプト作成モード

### 対象テーマ

病名×漢方テーマ

### 入力テーマ

線維筋痛症の漢方

### 目的

Decision Ledger / Answer Ledger の実装後に、ユーザーの軌道修正メモが最終プロンプトへ正しく反映されることを確認する。

特に、古い仮置きや前Stepの引き継ぎよりも、ユーザーが後から明示した確定条件を優先できるかを確認する。

### 途中のユーザー軌道修正

- 資料は漢方医向け
- 内部資料のため外部に出すことなし
- 3は初回から
- 専門度 上級者向け

### 期待される Answer Ledger

直前の確認質問に対応して、少なくとも以下の意味で記録されること。

- Q1: 読者に関する確認質問
  - A1: 漢方医向け
- Q2: 外部公開・利用場面に関する確認質問
  - A2: 内部資料。外部には出さない
- Q3: 処方・生薬・証・症状クラスターを初回に含めるか、後続調査に回すかの確認質問
  - A3: 初回から含める

質問文そのものはStep出力によって揺れてよい。
ただし、番号回答が質問文と紐づき、Decision Ledgerへ反映されること。

### 期待される Decision Ledger

- 主読者: 漢方医
- 副読者: 薬剤師 / 漢方相談員 / 漢方薬局スタッフ / 内部研修担当者
- 専門度: 上級者向け
- 用途: 内部学習・相談準備用
- 外部公開の有無: 外部公開しない
- 初回調査範囲: 医学的基礎 / 安全確認 / 症状クラスター / 漢方医学的病態 / 代表処方群・生薬候補の概観
- 後続調査に回す項目: 方剤別PMDA・添付文書照合 / 症例報告網羅 / エビデンス評価
- 除外範囲: 患者配布 / Web公開 / 販売促進 / 病名処方 / 処方ランキング / 服薬変更指示

### 期待される Final QA

- 古い仮置きの「漢方相談員・薬剤師向け」が主読者として残らない
- 最終プロンプトの主読者が「漢方医」になる
- 副読者が薬剤師、漢方相談員、漢方薬局スタッフ、内部研修担当者として分離される
- 専門度が「上級者向け」になる
- 用途が「内部学習・相談準備用」になる
- 外部公開、患者配布、Web公開、販売促進が除外される
- 初回調査に以下が含まれる
  - 医学的基礎
  - 安全確認
  - 症状クラスター
  - 漢方医学的病態
  - 代表処方群・生薬候補の概観
- 後続調査に以下が残る
  - 方剤別PMDA・添付文書照合
  - 症例報告網羅
  - エビデンス評価
- `## 矛盾検出` で、古い仮置きとDecision Ledgerが矛盾する場合はDecision Ledgerを優先して修正される
- 回答済みの質問を `## ユーザーへの確認質問` で再質問しない

### 手動確認手順

1. Deep Researchプロンプト作成モードを選ぶ。
2. 入力テーマに「線維筋痛症の漢方」を入れる。
3. Stepを進め、確認質問が出た後の軌道修正メモに以下を入れる。
   - 資料は漢方医向け
   - 内部資料のため外部に出すことなし
   - 3は初回から
   - 専門度 上級者向け
4. 次Stepのプロンプトで `## 回答済み質問 / Answer Ledger` と `## 確定済み条件 / Decision Ledger` を確認する。
5. Final QAの完成プロンプトで期待条件を確認する。

### 失敗とみなす条件

- 最終プロンプトの主読者が「漢方相談員・薬剤師向け」のまま残る
- 専門度「上級者向け」がDecision Ledgerまたは最終プロンプトに反映されない
- 「3は初回から」が無視され、処方・生薬・証・症状クラスターが後続調査だけに回る
- 外部公開、患者配布、Web公開、販売促進が除外されない
- Answer Ledgerが番号回答と直前質問を対応づけない
- Decision LedgerとFinal QAが矛盾しているのに `## 矛盾検出` が「なし」になる
